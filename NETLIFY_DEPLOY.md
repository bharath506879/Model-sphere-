# Netlify Deployment Guide

## Prerequisites
- GitHub account with the repository pushed
- Netlify account (free tier available at netlify.com)

## Quick Deploy Steps

### 1. Connect GitHub to Netlify
1. Go to [netlify.com](https://netlify.com)
2. Click "Add new site" → "Import an existing project"
3. Select GitHub and authorize Netlify
4. Choose the `bharath506879/Model-sphere-` repository
5. Click "Deploy site"

### 2. Build Settings
- **Build command:** `npm run build`
- **Publish directory:** `build`
- **Node version:** 18.x

### 3. Environment Variables (if needed)
If your backend is on a different server, add:
- Key: `REACT_APP_API_URL`
- Value: `https://your-backend-url`

### 4. Deployment
Netlify will automatically:
- Build your React app
- Apply `_redirects` rules for SPA routing
- Deploy to production

## Frontend Features Ready for Deployment ✓
- ✓ No login page (direct dashboard access)
- ✓ React 18 with React Router v6
- ✓ Responsive UI with Tailwind CSS
- ✓ SPA routing configured
- ✓ API integration ready

## Backend Integration
The frontend connects to backend at: `http://127.0.0.1:8000/api`

For production, update the API URL in your Netlify environment variables.

## Files Deployed
- `public/` - Static assets
- `src/` - React components
- `build/` - Generated by build process
- `netlify.toml` - Netlify configuration
- `public/_redirects` - SPA routing rules
